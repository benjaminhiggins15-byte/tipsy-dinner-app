# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Tipsy Dinner is a personal recipe management app built with TanStack Start (React + TanStack Router), designed for mobile-first experiences. The app uses localStorage for data persistence and is deployable to Cloudflare Workers.

## Development Commands

**Package Manager**: This project uses **Bun** (not npm/yarn/pnpm).

```bash
# Start development server
bun dev

# Build for production
bun build

# Build in development mode (with dev optimizations)
bun build:dev

# Preview production build
bun preview

# Lint code
bun lint

# Format code
bun format
```

## Architecture

### Stack
- **Framework**: TanStack Start (React 19 + TanStack Router)
- **Build Tool**: Vite 7 with @lovable.dev/vite-tanstack-config
- **Styling**: Tailwind CSS v4 with shadcn/ui components
- **Deployment**: Cloudflare Workers (configured via wrangler.jsonc)
- **Data Persistence**: Browser localStorage (migrating to Supabase)

### Directory Structure

```
src/
├── tipsy/              # Main application logic
│   ├── App.tsx         # Root app component with screen navigation
│   ├── data.ts         # Data layer: localStorage CRUD for recipes/categories
│   ├── Profile.tsx     # User profile management
│   ├── Onboarding.tsx  # First-time user experience
│   ├── AddYourOwn.tsx  # Add/edit recipe screen
│   └── NewCategory.tsx # Category creation/editing
├── routes/             # File-based routing (TanStack Router)
│   ├── __root.tsx      # Root route with layout & metadata
│   └── index.tsx       # Home route (renders App.tsx)
├── components/ui/      # shadcn/ui component library (Radix UI primitives)
├── lib/utils.ts        # Utility functions (cn, etc.)
├── hooks/              # React hooks
├── router.tsx          # Router configuration with error boundary
├── routeTree.gen.ts    # Auto-generated route tree (DO NOT EDIT)
└── styles.css          # Global styles
```

### Key Architecture Patterns

**Navigation**: The app uses a custom screen-based navigation system (not React Router navigation). The `App.tsx` component manages a stack-based navigation with `push()` and `back()` functions. Screen state is defined by the `Screen` discriminated union type.

**Data Layer**: All data operations go through `src/tipsy/data.ts`:
- `Recipe` and `SavedRecipe` types define the data model
- Categories have gradients and can be custom (user-created) or default
- Functions like `saveRecipe()`, `updateSavedRecipe()`, `deleteSavedRecipe()` handle persistence
- Custom categories are managed via `saveCustomCategory()`, `updateCustomCategory()`, `deleteCustomCategory()`

**Styling**: 
- Mobile-first design (320x640 viewport)
- Uses Tailwind CSS v4 utilities
- shadcn/ui components for UI primitives
- Custom color gradients for category cards

**Build Configuration**: The vite.config.ts uses `@lovable.dev/vite-tanstack-config` which bundles multiple plugins (React, TanStack Start, Tailwind, Cloudflare, etc.). Do not manually add these plugins or the build will break.

## Working with Routes

Routes are file-based in `src/routes/`. The route tree is auto-generated by `@tanstack/router-plugin` into `routeTree.gen.ts` during development. After adding/modifying routes, the dev server will regenerate this file automatically.

Example route structure:
```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/path")({
  component: YourComponent,
});
```

## Component Development

**UI Components**: Use existing shadcn/ui components from `src/components/ui/`. These are Radix UI primitives with Tailwind styling. To add new shadcn components, typically you'd run a command like `npx shadcn@latest add <component>`, but verify the project's preferred method first.

**Styling**: Use Tailwind utility classes. The project uses CVA (class-variance-authority) for component variants.

## Data Persistence

The app stores all data in localStorage:
- **Recipes**: Stored under key `"tipsyDinnerRecipes"` as `SavedRecipe[]`
- **Categories**: Stored under key `"tipsyDinnerCategories"` as `CustomCategory[]`
- **Onboarding**: `"tipsyDinnerOnboardingComplete"` (boolean)
- **User Profile**: `"tipsyDinnerPalate"`, `"tipsyDinnerInspiration"`, `"tipsyDinnerConstraints"`

All localStorage operations check for `typeof window === "undefined"` for SSR safety (TanStack Start has server-side rendering).

## Testing UI Changes

After making frontend changes, start the dev server with `bun dev` and test in a browser. The app renders a 320x640 mobile viewport centered on the page for easier desktop testing.

## Deployment

The app is configured for Cloudflare Workers deployment via `wrangler.jsonc`. The build output is handled by TanStack Start's server entry point. Planned deployment target: Vercel.

## Product Brief: Tipsy Dinner

### App Overview
**App name**: Tipsy Dinner — a personal digital cookbook named after the tradition of cooking with wine with my wife.

**Concept**: A personal recipe logging app that evolves with you. Starts as a beautiful digital cookbook of your original recipes, layered over time with AI that learns your cooking style to help craft new ones.

**Target user**: Myself first. Elevated home cook, appreciates really good food, cooks for two (wife) and for crowds (hosts Christmas etc). No central recipe hub currently — this solves that. Potential to scale to other users if the concept works well.

### Aesthetic Direction
Coastal European minimalist. Think Amalfi meets St. Petersburg FL. Cool, soft blues. Warm but not stuffy. References: Bon Appétit, Intermezzo coffee shop (intermezzo.co), restaurant Pulpo in St. Pete. Elevated but approachable.

**Color palette**:
- Background: `#EEF4F8`
- Accent blue: `#E6F1FB`
- Border: `#85B7EB`
- Dark navy: `#042C53`
- Mid blue: `#185FA5`
- Deep button blue: `#0C447C`

**Typography**: Serif headings and italic subtitles in Playfair Display. UI labels in DM Sans uppercase with letter spacing. Italic Playfair used for all supporting copy.

### The Three-Button Product Loop

1. **CRAFT / your next dish** — conversational AI recipe creation screen
2. **EXPLORE / your recipes** — saved recipe library
3. **CURATE / your menus** — saved recipes combined into occasion-based menus (not yet built)

### Craft Flow — AI Behavior (to be wired)

- Open-ended conversational AI prompt screen
- Knows user's palate, saved recipes, patterns
- Back-and-forth, follow-up questions, refinements
- When recipe is ready, mini player appears at bottom with recipe title
- User can expand full recipe card and save to Explore
- To refine, user continues typing in chat — no separate refine button
- "Write your own" button top right — disappears once conversation begins

### Craft System Prompt Rules

- Only update the recipe card and mini player title when user is explicitly iterating on the recipe — never on conversational tangents
- Answer wine pairing, technique, and other questions conversationally without touching the recipe card
- Tailor suggestions to user's palate, inspiration sources, and constraints at all times
- System prompt assembled from: base instructions + user profile (three onboarding answers) + live conversation

### AI Layer

- **Model**: claude-sonnet-4-5
- **Streaming**: Yes — token by token for conversational feel
- **API key**: Stored in `.env` as `ANTHROPIC_API_KEY`

### Data Layer

**localStorage keys**:
- `tipsyDinnerRecipes`
- `tipsyDinnerCategories`
- `tipsyDinnerOnboardingComplete`
- `tipsyDinnerPalate`
- `tipsyDinnerInspiration`
- `tipsyDinnerConstraints`

**Next**: Supabase migration (after AI is wired)

### Navigation Principles

- No text labels on any back arrow — icon only
- Back stack after saving always follows Explore hierarchy
- Craft screen never in back stack after recipe saved
- All screen transitions slide left/right

### Recipe Card Format — Universal

One consistent format everywhere: Hero gradient → category label → title → short description → tabbed view. Ingredients: name left, quantity right, dividing lines. Steps: large serif step number in muted blue (#85B7EB), step text alongside.
